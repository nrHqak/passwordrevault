# Руководство по тестированию приложения "КриптоСейф"

Это руководство поможет вам протестировать функциональность приложения "КриптоСейф" (Password Vault), разработанного на C# WinForms.

## 1. Подготовка среды

Для сборки и запуска приложения вам потребуется операционная система Windows и установленный .NET SDK (версия, совместимая с WinForms, обычно .NET Framework или .NET Core с поддержкой WinForms).

1.  **Клонируйте репозиторий GitHub:**
    Если вы еще не клонировали репозиторий, сделайте это:
    ```bash
    git clone https://github.com/nrHqak/passwordrevault.git
    cd passwordrevault
    ```

2.  **Откройте проект в Visual Studio:**
    Откройте файл `PasswordVault.sln` (или создайте новый проект WinForms и добавьте все файлы в него) в Visual Studio. Убедитесь, что все файлы `.cs` и `.Designer.cs` находятся в папке `PasswordVault` внутри вашего проекта.

3.  **Соберите проект:**
    В Visual Studio выберите `Build > Build Solution` (Сборка > Собрать решение).

4.  **Запустите приложение:**
    Нажмите `Start` (или F5) в Visual Studio, чтобы запустить приложение.

## 2. Тестирование функциональности

### 2.1. Форма входа (LoginForm)

1.  **Запуск приложения:** При первом запуске должно появиться окно входа.
2.  **Неверный пароль:** Введите любой текст, кроме "admin123", в поле "Мастер-пароль" и нажмите "ВОЙТИ".
    *Ожидаемый результат:* Должно появиться сообщение об ошибке: "Неверный пароль!".
3.  **Верный пароль:** Введите "admin123" в поле "Мастер-пароль" и нажмите "ВОЙТИ" (или нажмите Enter).
    *Ожидаемый результат:* Форма входа должна скрыться, и должна появиться главная форма (MainForm).

### 2.2. Главная форма (MainForm)

#### 2.2.1. Добавление новой записи

1.  На главной форме нажмите кнопку "+ ДОБАВИТЬ НОВУЮ ЗАПИСЬ" внизу.
    *Ожидаемый результат:* Должна открыться форма "Добавить / Редактировать запись" (AddEntryForm).
2.  **Заполнение полей:** Введите данные в поля "Название сервиса", "Логин" и "Пароль".
3.  **Генерация пароля (опционально):**
    *   Измените "Длину" пароля.
    *   Отметьте или снимите флажки "Буквы (A-Z)", "Цифры (0-9)", "Спецсимволы (!@#)".
    *   Нажмите кнопку "СГЕНЕРИРОВАТЬ".
    *Ожидаемый результат:* В поле "Пароль" должен появиться сгенерированный пароль, соответствующий выбранным критериям.
4.  **Сохранение записи:** Нажмите кнопку "СОХРАНИТЬ".
    *Ожидаемый результат:* Форма добавления/редактирования должна закрыться, и новая запись должна появиться в таблице на главной форме.
5.  **Отмена добавления:** Откройте форму добавления, но вместо сохранения нажмите "ОТМЕНА".
    *Ожидаемый результат:* Форма должна закрыться без добавления новой записи.

#### 2.2.2. Просмотр, копирование, изменение и удаление записей

1.  **Просмотр пароля:** Для существующей записи нажмите кнопку "Показать" в соответствующей строке.
    *Ожидаемый результат:* Должно появиться всплывающее окно с расшифрованным паролем для этой записи.
2.  **Копирование пароля:** Для существующей записи нажмите кнопку "Копировать" в соответствующей строке.
    *Ожидаемый результат:* Должно появиться сообщение "Скопировано!", и пароль должен быть скопирован в буфер обмена (вы можете вставить его в текстовый редактор, чтобы проверить).
3.  **Изменение записи:** Для существующей записи нажмите кнопку "Изменить" в соответствующей строке.
    *Ожидаемый результат:* Должна открыться форма "Добавить / Редактировать запись" с предзаполненными данными выбранной записи. Измените некоторые поля и нажмите "СОХРАНИТЬ". Запись в таблице должна обновиться.
4.  **Удаление записи:** Для существующей записи нажмите кнопку "Удалить" в соответствующей строке.
    *Ожидаемый результат:* Должно появиться окно подтверждения: "Удалить запись для [Название сервиса]?". Нажмите "Да". Запись должна исчезнуть из таблицы.

#### 2.2.3. Поиск записей

1.  Введите текст в поле "🔍 Поиск сервиса..." над таблицей.
    *Ожидаемый результат:* Таблица должна динамически фильтроваться, показывая только те записи, "Название сервиса" которых содержит введенный текст (без учета регистра).
2.  Очистите поле поиска.
    *Ожидаемый результат:* Должны отобразиться все записи.

### 2.3. Проверка сохранения данных

1.  Добавьте несколько записей.
2.  Закройте приложение (через крестик в углу).
3.  Запустите приложение снова и войдите с мастер-паролем "admin123".
    *Ожидаемый результат:* Все добавленные ранее записи должны быть загружены и отображены в таблице.

## 3. Проверка локализации (Русский язык)

Убедитесь, что весь текст в интерфейсе (заголовки форм, подписи к полям, кнопки, сообщения об ошибках) отображается на русском языке, как указано в требованиях.
